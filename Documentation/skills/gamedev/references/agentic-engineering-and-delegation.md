# Agentic Engineering and Delegation

## Use This Reference When

- The task involves externalizing expert knowledge into agents, skills, workflows, or documentation
- You need to separate intent, architecture, taste, and review from the mechanical execution work
- The problem can be decomposed into parallel work streams without losing coherence

## Expert Stance

Agentic engineering is a new multiplier.

The goal is to externalize expert judgment so a human expert can behave more like a leader, architect, reviewer, and taste-maker while agents handle large amounts of parallel execution. The leverage comes from clarity: separating the work from the result extraordinarily clearly.

This is not about abandoning standards. It is about turning hard-won domain knowledge into reusable rails, decision packets, review standards, and task decomposition.

## Core Rules

- Separate intent from implementation before delegating work.
- Keep the human in guidance, architecture, taste, prioritization, and review.
- Externalize expert judgment into reference files, constraints, examples, and acceptance criteria.
- Split work into parallelizable streams only when ownership and interfaces are clear.
- Use agents to scale execution, not to hide unclear thinking.
- Preserve decision quality by making success criteria concrete and reviewable.

## Practical Heuristics

- If a task cannot be described clearly enough to hand to an agent, it is probably not understood well enough yet.
- If multiple agents would step on the same files or concepts, refine the decomposition first.
- If the work needs specialist taste, keep the decision local and delegate the repetitive execution around it.
- If a pattern keeps recurring across tasks, encode it into the skill or supporting docs instead of re-explaining it every time.

## High-Signal Quotes

- "The next specialty that I'm building is my expertise in agentic and AI systems."
- "If I can take everything I know and externalize that into knowledge in agents, then I can effectively turn myself into the team that I wish I was."
- "I aim for is to be able to do the work of ten people."

## Common Mistakes

- Delegating before the architecture, goal, or acceptance criteria are clear
- Treating agentic workflows like a replacement for expertise instead of a multiplier on top of it
- Keeping important judgment in one person's head instead of encoding it into reusable guidance
- Using agents for noisy busywork without changing the quality of thinking or decomposition
